# 08. 운영 지표·루틴·로드맵

## 1. 운영 지표

부업형 구매대행은 매출만 보면 안 된다. 다음 지표를 함께 본다.

## 2. 상품 지표

| 지표 | 의미 |
|---|---|
| 등록 SKU 수 | 실제 운영 가능한 상품 수 |
| 후보 대비 등록률 | 소싱 품질 |
| 등록 대비 주문률 | 상품 선택 품질 |
| 조회수 | 노출 성과 |
| 클릭률 | 상품명·대표이미지 성과 |
| 전환율 | 가격·상세페이지·신뢰도 성과 |
| 품절률 | 공급 안정성 |
| 가격변동률 | 유지관리 부담 |

## 3. 주문 지표

| 지표 | 의미 |
|---|---|
| 주문 수 | 실판매 규모 |
| 주문당 매출 | 객단가 |
| 주문당 최종 마진 | 실질 수익성 |
| 발주 실패율 | 품절·가격변동 문제 |
| 평균 배송일 | 고객 만족도 |
| 배송 지연률 | CS 발생 가능성 |
| 반품률 | 상품 적합성 |
| 문의율 | 상세페이지/고지 부족 여부 |

## 4. 운영 지표

| 지표 | 의미 |
|---|---|
| 상품 1개 등록 소요 시간 | 자동화 가치 판단 |
| 주문 1건 처리 시간 | 운영 병목 판단 |
| CS 1건 처리 시간 | 템플릿 필요성 판단 |
| 주간 투입 시간 | 부업 지속 가능성 |
| 시간당 순이익 | 부업 효율 |
| 문제 상품 비율 | 카테고리 선택 품질 |

## 5. 일일 루틴

```markdown
- [ ] 신규 주문 확인
- [ ] 고객 정보 누락 확인
- [ ] 해외 재고·가격 변동 확인
- [ ] 발주 필요 주문 처리
- [ ] 배대지 입고 상태 확인
- [ ] 배송 지연 주문 확인
- [ ] 고객 문의 답변
- [ ] 문제 주문 메모
```

## 6. 주간 루틴

```markdown
- [ ] 신규 상품 후보 10~20개 수집
- [ ] 상품 후보 리스크 검토
- [ ] 등록 후보 3~5개 선정
- [ ] 상세페이지 작성/수정
- [ ] 기존 상품 가격·재고 확인
- [ ] 주문별 손익 정리
- [ ] CS 발생 원인 분류
- [ ] 판매 중단 상품 결정
- [ ] 다음 주 개선 과제 1~3개 선정
```

## 7. 월간 루틴

```markdown
- [ ] 월 매출 정리
- [ ] 월 순이익 정리
- [ ] 카테고리별 성과 비교
- [ ] 반품·환불 비용 정리
- [ ] 배송 지연률 확인
- [ ] 세무 증빙 정리
- [ ] 고위험 상품 제거
- [ ] 자동화 후보 작업 선정
- [ ] P0 서비스 요구사항 업데이트
```

## 8. 0~2주차 로드맵: 사업 준비

목표:

- 법적·운영 기본 세팅
- 판매채널 1개 선택
- 원가 시트 작성
- 금지/주의 품목 리스트 작성

작업:

```markdown
- [ ] 사업자등록 검토
- [ ] 통신판매업 신고 검토
- [ ] 판매채널 선택
- [ ] 배대지 선택
- [ ] 원가·마진 시트 작성
- [ ] CS 템플릿 초안 작성
- [ ] 제외 카테고리 정의
- [ ] 상품 후보 시트 작성
```

## 9. 3~4주차 로드맵: 상품 테스트

목표:

- 30~50개 상품 후보 검토
- 10~20개 상품 등록
- 실제 주문 발생 여부 확인

작업:

```markdown
- [ ] 상품 후보 수집
- [ ] 상품 리스크 점수화
- [ ] 원가·마진 계산
- [ ] 상세페이지 작성
- [ ] 상품 등록
- [ ] 가격·재고 모니터링
- [ ] 첫 주문 처리
```

## 10. 2개월차 로드맵: 운영 안정화

목표:

- 주문 처리 SOP 안정화
- CS 템플릿 개선
- 문제 상품 제거
- 수익성 높은 카테고리 파악

작업:

```markdown
- [ ] 주문별 처리 로그 정리
- [ ] 배송 지연 대응 개선
- [ ] 반품/환불 사례 정리
- [ ] 상세페이지 고지문 개선
- [ ] 가격 경쟁력 낮은 상품 제거
- [ ] 판매 가능성 높은 상품군 확장
```

## 11. 3개월차 로드맵: 자동화 후보 결정

목표:

- 실제 반복 작업 중 자동화 가치가 큰 영역 선정
- 상세페이지 생성 AI P0 명세 확정

작업:

```markdown
- [ ] 상품등록 소요 시간 측정
- [ ] 상세페이지 작성 소요 시간 측정
- [ ] CS 유형 빈도 분석
- [ ] 리스크 검토 누락 사례 분석
- [ ] 자동화 후보 우선순위 결정
- [ ] Codex 구현 범위 정의
```

## 12. P0 개발 로드맵

### Phase 1: 내부 템플릿

- Google Sheets 원가표
- 상품 검증 체크리스트
- 상세페이지 프롬프트
- CS 템플릿

### Phase 2: 로컬/간단 웹앱

- 상품 정보 입력폼
- 원가 계산
- 리스크 체크
- 상세페이지 초안 생성
- 고지문 생성

### Phase 3: 반자동 운영 보조

- 상품별 기록 DB
- 등록 전 체크리스트
- CS 템플릿 추천
- 판매 성과 기록
- 개선 메모 축적

### Phase 4: 서비스화 검토

- 사용자 계정
- 저장 기능
- 템플릿 관리
- 유료화
- 외부 판매채널 연동은 후순위
