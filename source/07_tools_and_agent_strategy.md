# 07. 외부 툴 활용과 자체 서브 에이전트 전략

## 1. 기본 판단

부업형 구매대행 초기에는 자체 서브 에이전트를 먼저 만들기보다, 외부 툴과 수동 SOP를 조합해 실제 업무 흐름을 검증하는 것이 낫다.

이유는 다음과 같다.

- 아직 반복 작업의 정확한 형태가 확정되지 않았다.
- 구매대행은 예외 상황이 많다.
- 개인정보와 통관 정보가 포함된다.
- 플랫폼 정책과 법규가 바뀔 수 있다.
- 자동화 오류가 곧 CS와 금전 손실로 이어질 수 있다.
- 초반에는 개발보다 운영 학습이 더 중요하다.

## 3. 외부 툴을 먼저 써야 하는 영역

### 3.1 원가·마진 계산

초기에는 Google Sheets 또는 Excel이 가장 적합하다.

이유:

- 구조가 단순하다.
- 계산식 수정이 쉽다.
- 직접 손익 구조를 이해할 수 있다.
- 자동화보다 검산이 중요하다.
- 세무 기록의 기초 자료로 활용 가능하다.

자체 에이전트는 나중에 시트 입력을 보조하는 수준이면 충분하다.

### 3.2 주문·정산 기록

초기에는 수동 기록이 필요하다.

이유:

- 주문 예외가 많다.
- 플랫폼별 정산 구조가 다르다.
- 세무 증빙과 연결된다.
- 자동화 오류가 치명적이다.

### 3.3 배송추적

초기에는 배대지 시스템과 택배 조회를 그대로 사용한다.

자체 자동화는 주문량이 늘어난 뒤에 고려한다.

## 4. 자체 서브 에이전트 가치가 큰 영역

### 4.1 상품 리스크 검토 에이전트

입력:

- 상품 URL
- 상품명
- 카테고리
- 이미지 설명
- 원본 상세정보

출력:

- 금지/주의 품목 가능성
- KC·식약처·의료기기·어린이제품 의심 여부
- 브랜드/IP 리스크
- 배송 제한 요소
- 초보자 취급 가능 등급
- 확인해야 할 공식 자료 목록

가치:

- 초보자가 놓치기 쉬운 리스크를 사전에 알려준다.
- P0에서 매우 중요하다.
- 단, 최종 법적 판단을 대신해서는 안 된다.

### 4.2 상세페이지 생성 에이전트

입력:

- 상품 정보
- 타깃 고객
- 경쟁 상품 메모
- 배송/통관 조건
- 반품 조건

출력:

- 상품명 후보
- 상세페이지 구조
- 스펙표
- 구매 전 확인사항
- 고지문
- CS 예상 질문

가치:

- 반복 작업 시간을 크게 줄인다.
- 서비스화 가능성이 가장 높다.
- 사용자가 직접 체감하기 쉽다.

### 4.3 CS 템플릿 에이전트

입력:

- 상황 유형
- 주문 상태
- 고객 문의 내용
- 배송 상태

출력:

- 고객 응대문 초안
- 사과/안내/선택지 제시 문구
- 내부 처리 체크리스트

가치:

- 부업 운영자의 피로도를 줄인다.
- 단, 자동 발송은 초기에는 금지하고 초안 생성에 그친다.

### 4.4 상품 소싱 보조 에이전트

입력:

- 관심 카테고리
- 제외 조건
- 목표 마진
- 소싱처
- 국내 경쟁 링크

출력:

- 후보 상품 정리
- 비교표
- 리스크 메모
- 판매 가능성 가설

가치:

- 장기적으로 중요하다.
- 하지만 초기에는 크롤링·플랫폼 약관·데이터 품질 문제가 있어 수동+반자동이 적합하다.

## 5. 외부 툴 vs 자체 에이전트 비교

| 기준 | 외부 툴 | 자체 서브 에이전트 |
|---|---|---|
| 초기 도입 속도 | 빠름 | 느림 |
| 비용 | 낮거나 월 구독 | 개발 시간 비용 큼 |
| 유연성 | 제한적 | 높음 |
| 운영 이해도 향상 | 높음 | 잘못 만들면 낮음 |
| 자동화 수준 | 중간 | 높음 |
| 오류 책임 | 사용자가 검수 | 사용자가 설계 책임 |
| 개인정보 리스크 | 툴별 상이 | 설계에 따라 통제 가능 |
| 서비스화 가능성 | 낮음 | 높음 |
| P0 적합성 | 운영 검증에 적합 | 특정 병목에 한해 적합 |

## 6. 개발 우선순위

### 1순위: 상세페이지 생성 + 고지문 생성

가장 직접적인 시간 절감 효과가 있다.

### 2순위: 상품 리스크 체크리스트

구매대행 초보자의 실패 가능성을 낮춘다.

### 3순위: 원가·마진 계산 시트 자동 보조

정확한 사업 판단에 필요하다.

### 4순위: CS 템플릿 생성

주문 발생 후 운영 피로도를 줄인다.

### 5순위: 상품 후보 수집 자동화

중요하지만 초기에는 과도한 자동화 위험이 있다.

## 7. P0 기능 구성안

```text
상품 URL/정보 입력
→ 상품 기본정보 정리
→ 금지·주의 카테고리 체크
→ 원가·마진 계산 입력폼
→ 상세페이지 초안 생성
→ 해외구매대행 고지문 생성
→ CS 템플릿 생성
→ 등록 전 체크리스트 출력
```

## 8. 하지 말아야 할 자동화

초기에는 다음 자동화를 피한다.

- 고객 개인정보 자동 수집·저장
- 개인통관고유부호 외부 AI 입력
- 자동 발주
- 자동 환불 판단
- 고위험 품목 자동 추천
- 오픈마켓 대량 자동 업로드
- 법적 판단을 확정적으로 내려주는 기능
