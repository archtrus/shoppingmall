# 01. 프로젝트 정의와 P0 범위

## 1. 프로젝트 목적

현재 프로젝트의 최종 목적은 **해외 구매대행 셀러를 위한 상세페이지 생성 AI 및 운영 보조 서비스**를 만드는 것이다.

다만 바로 서비스를 만들기보다, 사용자가 직접 부업 형태로 해외 구매대행을 작게 운영하면서 다음을 검증한다.

- 실제 구매대행 업무가 어떤 흐름으로 이루어지는지
- 어떤 단계가 반복적이고 자동화 가치가 큰지
- 어떤 데이터가 상세페이지 생성과 전환율 개선에 필요한지
- 개인 셀러가 돈을 낼 만한 실질적 페인포인트가 무엇인지
- AI가 잘할 수 있는 일과 사람이 직접 판단해야 하는 일을 어떻게 나눌지

## 2. 현재 P0 정의

P0는 “완전한 구매대행 자동화 시스템”이 아니다.

현재 P0는 다음에 가깝다.

> 개인이 부업형 해외 구매대행을 안전하게 시작하고, 상품 후보를 검토한 뒤, 판매 가능성이 있는 상품에 대해 상세페이지 초안과 운영 체크리스트를 생성하는 내부 보조 시스템.

## 3. P0에서 해결할 핵심 문제

### 3.1 상품을 어떻게 소싱할지 모르는 문제

초기 사용자가 가장 먼저 막히는 지점은 “어떤 상품을 어디서 어떻게 찾는가”이다.

따라서 P0는 단순히 상세페이지를 예쁘게 만드는 도구가 아니라, 최소한 다음을 지원해야 한다.

- 상품 후보 수집
- 카테고리 리스크 필터링
- 국내 판매 가능성 판단
- 원가·마진 계산
- 등록 전 체크리스트 생성
- 상세페이지 초안 생성

### 3.2 구매대행 운영 흐름을 모르는 문제

해외 구매대행은 상품을 올리는 일보다 주문 이후의 운영이 중요하다.

- 주문 확인
- 고객 정보 확인
- 해외 사이트 발주
- 배대지 입력
- 통관 정보 확인
- 국내 배송 추적
- 지연·품절·파손·환불 대응
- 정산·세무 기록

P0는 이 흐름을 전부 자동화하지는 않더라도, 각 단계의 누락을 줄이는 체크리스트와 기록 체계를 제공해야 한다.

### 3.3 상세페이지 품질과 고지 누락 문제

구매대행 상세페이지는 일반 쇼핑몰 상세페이지와 다르다.

반드시 다음 내용을 명확히 고지해야 한다.

- 해외 구매대행 상품임
- 배송 기간
- 통관 과정
- 개인통관고유부호 필요 여부
- 관부가세 부담 기준
- 반품·교환 비용과 조건
- 품절 가능성
- 색상·사이즈·옵션 오차 가능성
- 인증·안전 관련 제한 사항

P0의 상세페이지 생성기는 “마케팅 문구 생성기”가 아니라 **판매 전환 + 리스크 고지 + CS 예방**을 동시에 수행해야 한다.

## 4. P0에서 하지 않을 것

초기에는 다음 기능을 제외한다.

- 다중 오픈마켓 자동 등록
- 대량 SKU 크롤링과 자동 업로드
- 결제 기능
- 고객 개인정보 장기 저장
- 완전 자동 발주
- 자동 반품 처리
- 세무 신고 자동화
- 고위험 품목 추천
- 브랜드·가품 리스크가 큰 상품 추천
- 식품·건기식·화장품·의료기기·전기용품·어린이제품 중심 운영

## 5. 초기 타깃 사용자

초기 타깃은 다음과 같다.

- 부업으로 구매대행을 시작하려는 개인
- 대량 사입보다 무재고 주문형 운영을 선호하는 사람
- 스마트스토어·쿠팡·11번가 등 국내 오픈마켓 초보 셀러
- 상품등록과 상세페이지 작성에 시간을 많이 쓰는 사람
- 규제·통관·CS 리스크가 걱정되는 사람

## 6. 서비스 가설

### 핵심 가설

개인 구매대행 셀러는 단순 상세페이지 생성보다 다음 조합에 더 큰 가치를 느낀다.

> 상품 후보 검토 → 리스크 필터링 → 원가·마진 계산 → 상세페이지 초안 → 고지문/CS 템플릿 → 운영 체크리스트

### 검증해야 할 질문

- 개인 셀러가 실제로 가장 시간을 많이 쓰는 단계는 어디인가?
- 상세페이지 생성만으로 지불 의사가 생기는가?
- 상품 소싱/검증까지 포함해야 결제 가치가 생기는가?
- 셀러는 자동화보다 체크리스트형 보조를 더 신뢰하는가?
- 규제·통관·반품 고지를 AI가 정리해주는 기능에 비용을 지불할까?
