# 03. 해외 구매대행 운영 워크플로우 SOP

## 1. 전체 워크플로우

해외 구매대행 업무는 다음 10단계로 나눌 수 있다.

1. 시장·카테고리 선정
2. 상품 후보 수집
3. 상품 리스크 검토
4. 원가·마진 계산
5. 판매채널 등록 준비
6. 상세페이지·고지문 작성
7. 주문 접수
8. 해외 발주·배대지 처리
9. 통관·국내배송 추적
10. CS·정산·개선

## 2. 단계별 세부 작업

### 2.1 시장·카테고리 선정

목적은 “팔릴 만한 상품”을 찾기 전에 “하지 말아야 할 상품”을 먼저 제거하는 것이다.

작업 단위:

- 금지·제한 품목 리스트 확인
- KC·식약처·검역·의료기기·화장품 관련 여부 확인
- 브랜드·상표권 리스크 확인
- 파손·반품 가능성 확인
- 배송비가 과도한 부피/중량 상품 제외
- 국내 경쟁 판매가 확인
- 고객이 구매대행을 감수할 이유 확인

산출물:

- 카테고리 후보 리스트
- 제외 카테고리 리스트
- 카테고리별 리스크 메모

### 2.2 상품 후보 수집

작업 단위:

- 해외 플랫폼에서 상품 후보 수집
- 국내 오픈마켓에서 유사 상품 검색
- 가격대 비교
- 리뷰 수와 평점 확인
- 옵션 구조 확인
- 이미지·상세정보 확보 가능성 확인
- 배송 가능 국가와 배송 조건 확인

산출물:

- 상품 후보 시트
- 원본 URL
- 대표 이미지
- 해외 가격
- 예상 판매가
- 경쟁 상품 링크

### 2.3 상품 리스크 검토

작업 단위:

- 금지 품목 여부 확인
- 전기·전자 여부 확인
- 어린이 제품 여부 확인
- 식품·건기식·화장품·의료기기 여부 확인
- 유명 브랜드·캐릭터·로고 포함 여부 확인
- 배터리 포함 여부 확인
- 액체·분말·자석·칼날 등 배송 제한 요소 확인
- 과대광고 문구 여부 확인

판정:

| 판정 | 의미 |
|---|---|
| A | 초기 테스트 가능 |
| B | 추가 확인 후 가능 |
| C | 초보자 초기 제외 |
| D | 취급 금지 또는 매우 부적합 |

### 2.4 원가·마진 계산

작업 단위:

- 해외 상품가 입력
- 해외 현지 배송비 입력
- 배대지 비용 추정
- 국제배송비 추정
- 국내배송비 반영
- 플랫폼 수수료 반영
- 환율 버퍼 반영
- 반품·분실 리스크 비용 반영
- 목표 마진율 계산
- 경쟁 판매가와 비교

산출물:

- 상품별 손익표
- 최소 판매가
- 권장 판매가
- 마진율
- 가격 경쟁력 판정

### 2.5 판매채널 등록 준비

작업 단위:

- 상품명 작성
- 카테고리 매핑
- 옵션 구조 정리
- 대표 이미지 선정
- 상세 이미지 정리
- 배송비/반품비 설정
- 배송기간 고지
- 해외 구매대행 고지
- 개인통관고유부호 안내
- 관부가세 안내

산출물:

- 등록용 상품 정보
- 상세페이지 초안
- 고지문
- 옵션표

### 2.6 상세페이지·고지문 작성

상세페이지는 “구매 전환”과 “리스크 고지”를 동시에 수행해야 한다.

포함 항목:

- 상품 핵심 가치
- 사용 장면
- 주요 스펙
- 옵션별 차이
- 사이즈/색상 안내
- 해외 구매대행 안내
- 배송기간 안내
- 통관 안내
- 반품·교환 조건
- 품절·가격변동 가능성
- 고객 확인사항

### 2.7 주문 접수

작업 단위:

- 신규 주문 확인
- 결제 완료 여부 확인
- 고객명·연락처·주소 확인
- 개인통관고유부호 확인
- 옵션 확인
- 재고와 가격 재확인
- 고객에게 발주 예정 안내

주의점:

- 발주 전 해외 사이트 재고와 가격을 반드시 재확인한다.
- 옵션명이 국내 등록 정보와 일치하는지 확인한다.
- 고객 정보는 최소한으로만 취급한다.

### 2.8 해외 발주·배대지 처리

작업 단위:

- 해외 사이트 주문
- 배대지 주소 입력
- 주문번호 기록
- 트래킹번호 기록
- 배대지 입고 확인
- 국제배송 신청
- 통관 정보 입력
- 배송비 확정

산출물:

- 주문별 발주 기록
- 해외 주문번호
- 배대지 신청번호
- 국제배송 추적번호

### 2.9 통관·국내배송 추적

작업 단위:

- 통관 진행 상태 확인
- 보류·지연 여부 확인
- 국내 배송 전환 여부 확인
- 고객에게 배송 현황 안내
- 지연 시 사전 안내
- 배송 완료 확인

주의점:

- 통관 단계에서 고객 정보 오류가 있으면 지연될 수 있다.
- 배송 지연은 고객이 문의하기 전에 먼저 안내하는 편이 CS 비용을 줄인다.

### 2.10 CS·정산·개선

작업 단위:

- 배송 지연 문의 대응
- 품절 안내
- 옵션 오배송 대응
- 파손·불량 접수
- 반품 가능 여부 판단
- 환불 비용 계산
- 주문별 최종 손익 기록
- 문제 상품 제외
- 상세페이지 문구 개선

산출물:

- CS 로그
- 반품/환불 기록
- 상품별 개선 메모
- 판매 지속/중단 판단
